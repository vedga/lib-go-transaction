Предлагаемый фреймворк предоставляет возможность распределенного выполнения задач в транзакционном режиме. Также поддерживается outbox-паттерн для гарантированной отправки сообщения при фиксировании транзакции.

Основная идея заключается в следующем: все сервисы получают сообщения из единой шины данных. Если сервис умеет обрабатывать текущую задачу в очереди, то он запускает ее на выполнение. Если сервис не умеет обрабатывать текущую задачу, то полученное сообщение им игнорируется (ожидается, что задача будет обработана тем сервисом, который умеет это делать). Каждая задача во время выполнения также может формировать список задач для отката транзакции путем помещения их на стек отката через метод QueueRollbackTask() транзакции. Кроме того задача может положить произвольные данные на стек данных, которые могут быть в дальнейшем обработаны последующими задачами транзакции.

Во время выполнения задачи в ней может возникнуть восстановимая или не восстановимая ошибка. В случае восстановимой ошибки задача должна вернуть ошибку, созданную при помощи NewRetryTaskError(maxRetryCount). В этом случае исходная транзакция будет повторно помещена на шину данных, но в ней увеличится счетчик попыток выполнения текущей задачи. Счетчик попыток доступен для чтения только внутри метода Run() задачи путем вызова функции RunContext(context.Context). В случае возникновения невосстановимой ошибки задача должна вызвать метод Rollback(), который инициирует откат транзакции.

Номер повтора функция RunContext(context.Context).Attempt возвращает начиная с 0 (т.е. первая попытка вызова задачи вернет значение 0, вторая 1 и т.п.).

Для асинхронного выполнения задач можно по входным параметрам формировать контекст задачи и добавлять его в транзакцию методом QueueTask() или QueueEncodedTask().

Если задача заказывает откат транзакции, то задачи, которые включаются в откат этой задачей также будут выполняться. Если задача, заказавшая откат, возвращает код ошибки (кроме ошибки запроса повторов?), то транзакция немедленно завершается без выполнения отката.

Транзакция и ее методы не являются goroutine-safe имплементацией, это сделано специально для уменьшения накладных затрат при обработке задач.

Метод Run() задачи (интерфейс Task) может возвращать следующие коды ошибок:

| Код ошибки      | Назначение                                                                                                                                                                                                                                                                                                                   | Примечание                                                                                                                                          |
|-----------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------|
|(nil)            | Задача полностью выполнена, удалена из транзакции.                                                                                                                                                                                                                                                                           | Можно приступать к выполнению следующей задачи в транзакции.                                                                                        |
|RetryTaskError   | Требуется повтор выполнения этой задачи с исходным контекстом не более указанного кол-ва попыток.                                                                                                                                                                                                                            |                                                                                                                                                     |
|ErrMigrate       | Контекст текущей задачи будет сохранен в транзакции и сама транзакция будет помещена на шину сообщений.                                                                                                                                                                                                                      | Можно использовать для принудительной миграции выполнения задачи.                                                                                   |
|ErrNoAvailableTasks | В транзакции больше нет задач, готовых к выполнению.                                                                                                                                                                                                                                                                         | Может использоваться для реализации паттерна проектирования Outbox, при этом ответственность за продолжение обработки транзакции ложится на задачу. |
|другие ошибки    | Ошибка передается на верхний уровень. Если в этот момент шла обработка задач транзакции, то устанавливается индикатор отката транзакции и в качестве причины пишется содержимое ошибки. Если в этот момент выполнялся откат транзакции, то ошибка является невосстановимой и транзакция попадает в неопределенное состояние. |                                                                                                                                                     |


Любые ошибки, кроме **RetryTaskError** и **ErrMigrate** приводят к откату транзакции. Если в задаче, из-за которой производится откат транзакции, были добавлены операции для отката, то они также будут выполнены (фактически они выполняются в самом начале отката). Если транзакция уже находится в режиме отката, то эти ошибки считаются невосстановимыми и транзакция попадает в состояние "откат невозможен".

Метод Run транзакции (интерфейс Transaction) может возвращать следующие коды ошибок:
* **nil** - текущая задача транзакции была выполнена без ошибок и удалена из транзакции. Возможно в транзакции имеются еще задачи, готовые к выполнению.
* **ErrNoAvailableTasks** - в транзакции больше нет задач, которые можно выполнять и транзакцию можно успешно завершить.

Примеры использования можно найти в каталоге examples.

Потенциальное использование:
* Распределенная транзакция между независимыми сервисами с возможностью ее отката. Основное назначение по дизайну.
* Перенос выполнения долгих процессов между экземплярами сервиса в случае его остановки в момент переноса между нодами K8S. Идея заключается в том, что весь контекст долгой операции собран в контексте задачи. При получении от управляющей системы сигнала kill задача отправляется на шину сообщений в виде транзакции (при этом текущий контекст задачи сохраняется в транзакции), после чего она может быть принята к исполнению каким-либо другим живым подом сервиса.
* Распределенный таймер. По аналогии с предыдущим применением. В случае переноса будет некоторое ухудшение его точности в сторону увеличения на время обработки транзакции, но для больших временных интервалов проблема не выглядит как критичная.
