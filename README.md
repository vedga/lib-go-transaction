Предлагаемый фреймворк предоставляет возможность распределенного выполнения задач в транзакционном режиме. Также поддерживается outbox-паттерн для гарантированной отправки сообщения при фиксировании транзакции.
Основная идея заключается в следующем: все сервисы получают сообщения из единой шины данных. Если сервис умеет обрабатывать текущую задачу в очереди, то он запускает ее на выполнение. Если сервис не умеет обрабатывать текущую задачу, то полученное сообщение им игнорируется (ожидается, что задача будет обработана тем сервисом, который умеет это делать).
